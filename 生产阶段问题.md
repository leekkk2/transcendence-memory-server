# 生产阶段问题记录（Transcendence Memory Server）

> 目标：为后续开源收敛问题与修复方案，包含本次多模态（rag-everything / RAG-Anything）集成细节。

## 背景

- 生产部署为 Docker Compose
- 运行入口：`rags.zweiteng.tk`
- 上游 LLM/Embedding/VLM 统一走 `https://newapi.zweiteng.tk/v1`
- 模型：LLM = `gpt-5.4`，Embedding = `gemini-embedding-001`，VLM = `gpt-5.4`

## 问题 1：multimodal / rag-everything 未启用

### 现象
- `GET /health` 返回 `architecture = lancedb+lightrag`
- `modules.multimodal.package_available = false`
- `.env` 中已配置 `VLM_API_KEY`，但仍未启用 multimodal

### 根因
Dockerfile 中的 RAG-Anything 安装逻辑失败但被**静默吞掉**：

```bash
pip install --no-cache-dir git+https://github.com/HKUDS/RAG-Anything.git || true
```

具体问题：
1) builder stage 没安装 `git`，导致 `pip install git+...` 直接失败
2) `|| true` 屏蔽了失败，使镜像构建“表面成功”但功能缺失

### 修复方案（已验证）
1) builder stage 安装 `git`
2) 保持 RAG-Anything 安装逻辑，但**不要静默失败**

示例修复：
```dockerfile
FROM python:3.13-slim AS builder
RUN apt-get update && apt-get install -y --no-install-recommends git && rm -rf /var/lib/apt/lists/*
...
RUN pip install --no-cache-dir git+https://github.com/HKUDS/RAG-Anything.git
```

### 结果
- health 显示 `architecture = rag-everything`
- `multimodal.enabled = true`，`package_available = true`

## 问题 2：tm-server / connection token CLI 不可用

### 现象
- 容器内执行 `tm-server export-token` 报错：`executable file not found`
- `python -m tm_server` 提示模块缺失

### 根因
Dockerfile 只复制了源码与脚本，但**没有把项目本体安装进 Python 环境**，因此：
- console script 未注册
- 模块不可导入

### 修复方案（已验证）
在 build 阶段显式安装项目本体：

```dockerfile
COPY pyproject.toml README.md ./
COPY src/ ./src/
RUN pip install --no-cache-dir .
```

### 结果
- `tm-server export-token` 可用
- `python -m tm_server export-token` 可用

## 问题 3：RAG-Anything 安装依赖过重/隐性膨胀

### 现象
- 安装 `raganything` 会拉取大量依赖（含大型 torch/cuda 组件）
- ARM 机器上构建时间极长且镜像显著膨胀

### 原因说明
RAG-Anything 官方依赖链包含 MinerU / 视觉解析组件，属于**重依赖设计**，默认安装会拉取大量包。

### 优化建议（用于开源收敛）

#### 方案 A：将 RAG-Anything 设为可选安装
- 通过 build arg 或 compose profile 控制是否安装
- 例如：`INSTALL_RAGANYTHING=true/false`

#### 方案 B：使用官方 `pip` extras 管理
参考官方文档：

```bash
pip install raganything
pip install 'raganything[all]'
pip install 'raganything[image]'
pip install 'raganything[text]'
```

建议在 Dockerfile 中显式选用所需 extras，避免默认全量安装。

#### 方案 C：将 RAG-Anything 作为 optional dependency
在 `pyproject.toml` 内通过 `extras` 管理：

```toml
[project.optional-dependencies]
multimodal = ["raganything @ git+https://github.com/HKUDS/RAG-Anything.git"]
```

然后在容器中用：
```bash
pip install .[multimodal]
```

## 建议在开源 README/部署指南里新增的提醒

1) **RAG-Anything 安装依赖极重**，建议明确“可选安装”
2) **不要用 `|| true` 静默吞掉安装失败**
3) **Dockerfile 必须安装项目本体**，否则 CLI 无法用
4) ARM 环境注意构建时间/镜像大小，建议预构建或提供 slim profile
